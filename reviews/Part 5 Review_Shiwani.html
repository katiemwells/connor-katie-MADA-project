2.3 Data description

2.3.1 Feedback and Comments

The data is described well. There could still be more information included about the data source.

2.3.2 Summary assessment

  • source and overall structure of data somewhat explained

2.4 Data wrangling and exploratory analysis

2.4.1 Feedback and Comments

The data wrangling and exploratory analysis was done well. One suggestion could be creating a chunk of code for loading packages in each .qmd files. For example, while I tried running exploration.qmd file, I had an error saying I have no package installed for running a particular function used in that file. So in that case, it’s better if we could list all the required packages at the top of the file that are needed to run each of the particular files.

2.4.2 Summary assessment

  • essentially no weaknesses in wrangling and exploratory component

2.5 Appropriateness of Analysis

2.5.1 Feedback and Comments

Analysis section was easy to follow and appropriate. But I wish there were a little bit more detailed comments on code files. More ML model analysis parts could be added as well.

2.5.2 Summary assessment

  • strong and reasonable analysis

2.6 Presentation

2.6.1 Feedback and Comments

The results are presented well with some rooms for improvements in Tables. I know its hard trying to fit the table but you could play around with codes a little bit to fit the full analysis table in a doc file. Figures 1 and 2 look good but maybe you could log scale Y-axis. It will also be better to label the figures and tables as Table 1 and Figure 1 and so on.

2.6.2 Summary assessment

  • results are presented ok, with room for improvement

2.7 Discussion/Conclusions

2.7.1 Feedback and Comments

Summary is clear and concise. Have done a great job linking the study results. Strengths and limitations have been discussed well.

2.7.2 Summary assessment

  • strong, complete and clear discussion

2.8 Further comments

Overall, the research question is of contemporary type, and the study result turned out really interesting. Good job with well formulated hypothesis and following the analysis accordingly.

3 Overall project content evaluation

3.1 Structure

3.1.1 Feedback and Comments

Everything is structured well and was easy to follow along. It might not be necessary but I would suggest adding a readme file in each project folders so that we don’t always need to go the main readme file to find out what those folders are for.

3.1.2 Summary assessment

  • well structured

3.2 Documentation

3.2.1 Feedback and Comments

The documentation is thorough but I would suggest adding readme files as mentioned earliers in each project folders.

3.2.2 Summary assessment

  • fully and well documented

3.3 Reproducibility

3.3.1 Feedback and Comments

As mentioned earlier, the packages were not listed for each files so I had trouble running code files as I had to manually look for packages. So, I would suggest adding list of packages in each files.

3.3.2 Summary assessment

  • small parts not reproducible or required manual intervention

3.4 Thoroughness

3.4.1 Feedback and Comments

The study was thorough and easy to follow. But there are still some rooms for additions for list of packages and more ML model additions part.

3.4.2 Summary assessment

  • decent level of thoroughness

3.5 Further comments

Not major issues. But could add a project title, readme files for each folders, more ML models, change the table layout, and include list of packages. I really enjoyed reviewing this project. Overall, great job guys!!!
